## Project Overview
FitFindr is an intelligent, multi-step AI agent designed to streamline the thrifting process. Instead of just returning search results, FitFindr orchestrates a chain of specialized tools to find secondhand clothing, evaluate how new items integrate with a user's existing wardrobe, and generate highly shareable social media captions. The agent uses strict conditional branching and state management to ensure tools are only called when they have the required context, degrading gracefully when edge cases occur.

## Tool Inventory

### 1. `search_listings`
* **Purpose:** Searches a local mock dataset of secondhand clothing across multiple metadata fields (title, description, style tags, category).
* **Inputs:** * `description` (str): Keyword or style description (e.g., "vintage graphic tee").
  * `size` (str): Requested size (e.g., "M", "US 8"). Can be `None`.
  * `max_price` (float): Maximum price constraint. Can be `None`.
* **Outputs:** `list[dict]` containing matching listing objects.

### 2. `suggest_outfit`
* **Purpose:** Evaluates a newly found item against the user's existing wardrobe profile and generates a cohesive styling suggestion.
* **Inputs:** * `new_item` (dict): The selected item object returned from `search_listings`.
  * `wardrobe` (dict): The user's current wardrobe schema.
* **Outputs:** `str` (Natural language outfit recommendation).

### 3. `create_fit_card`
* **Purpose:** Generates a punchy, social-media-ready caption ("fit card") incorporating the item details and the outfit styling.
* **Inputs:** * `outfit` (str): The styling suggestion generated by `suggest_outfit`.
  * `new_item` (dict): The selected item object (used for price, brand, and platform details).
* **Outputs:** `str` (Formatted caption string).

## Architecture & The Planning Loop

The agent is driven by a conditional planning loop in `agent.py`. It does not execute a rigid sequence; instead, it reads the success state of previous steps before deciding to proceed.

1. **Parameter Extraction:** The user's query is parsed for search constraints.
2. **Search Execution:** `search_listings` is called.
3. **Conditional Branching:** * **Failure Path:** If the search returns `[]`, the loop halts immediately. It populates an error message and bypasses the LLM tools entirely to save compute and prevent hallucinations.
   * **Success Path:** If items are found, the top result is pushed into the session state, and the loop proceeds to the styling phase.
4. **Styling & Formatting:** The `suggest_outfit` and `create_fit_card` tools are called sequentially, strictly relying on the data passed down through the state object.

## State Management

State is managed via a shared `session` dictionary that persists throughout a single interaction.
* When a search succeeds, `results[0]` is assigned to `session["selected_item"]`.
* Subsequent tools pull directly from `session["selected_item"]` and `session["outfit_suggestion"]` rather than re-parsing the initial user prompt.
This approach prevents the LLM from losing context or hallucinating item details mid-chain.

---

## Error Handling & System Resilience

To ensure the agent behaves predictably in production, every tool has a deliberate failure mode that prevents app crashes and unhelpful blank states.

* **Empty Search Fallback:** When queried for a "$5 solid gold Rolex" or an "XXS designer ballgown," `search_listings` returns an empty list. The loop catches this and returns a friendly string: *"Sorry, I couldn't find any listings matching that. Try loosening the price, size, or describing the item a little differently."*
* **Empty Wardrobe Fallback:** If a new user with no saved clothes requests styling, `suggest_outfit` detects the empty `items` array and dynamically pivots to suggesting universal basics (e.g., "Pair this with classic blue jeans and white sneakers").
* **Broken Context Fallback:** If the styling tool fails to pass a valid string to `create_fit_card`, the fit card tool drops the outfit context and successfully generates a caption hyping up the individual item's price and platform.
